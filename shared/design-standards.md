# Agentic Assistants Design Standards

**Version:** 2.1.0  
**Status:** Canonical

## Purpose

This document defines the universal design principles inherited by every assistant responsible for creating visual assets, user interfaces, presentations, marketing materials, documents, and digital experiences.

The objective is to produce designs that are clear, intentional, consistent, and production-ready.

---

# Core Design Principles

Every design should be:

- Purposeful
- Simple
- Consistent
- Accessible
- Visually balanced
- Brand appropriate
- Easy to understand

Design exists to improve communication—not to add decoration.

---

# User First

Every visual decision should answer:

- Who is this for?
- What action should they take?
- What information matters most?
- What should draw their attention first?

Design should reduce cognitive load, not increase it.

---

# Visual Hierarchy

Strong designs guide attention intentionally.

Prioritize:

1. Primary message
2. Supporting information
3. Secondary details
4. Calls to action

The most important element should always be visually dominant.

---

# Layout

Prefer:

- Consistent spacing
- Grid-based alignment
- Generous whitespace
- Predictable structure
- Clean margins

Avoid clutter.

Every element should have a reason to exist.

---

# Typography

Typography should improve readability.

Prefer:

- Limited font families
- Clear hierarchy
- Appropriate contrast
- Comfortable line spacing
- Consistent sizing

Avoid decorative typography unless required by the brand.

---

# Color

Use color intentionally.

Color should:

- Reinforce hierarchy
- Support branding
- Improve usability
- Highlight actions
- Communicate status

Never rely on color alone to communicate meaning.

---

# Imagery

Images should be:

- Authentic
- Relevant
- High quality
- Consistent in style
- Appropriate for the audience

Avoid generic stock imagery whenever possible.

Prefer realistic photography over obviously AI-generated imagery unless the assignment specifically calls for stylized artwork.

---

# AI Image Standards

When creating AI-generated visuals:

- Prioritize realism.
- Preserve natural lighting.
- Maintain correct proportions.
- Avoid distorted anatomy.
- Avoid unrealistic text.
- Avoid fake interfaces.
- Avoid invented logos.
- Avoid artificial-looking hands.
- Preserve photographic quality whenever appropriate.

If text is required, it should be added during production unless image generation supports reliable typography.

---

# User Interface Standards

Interfaces should prioritize:

- Simplicity
- Predictability
- Accessibility
- Readability
- Fast recognition

Reduce the number of decisions required from the user.

---

# Presentation Standards

Slides should communicate one primary idea.

Prefer:

- One headline
- Supporting visual
- Minimal text
- Clear takeaway

Avoid slides that become documents.

---

# Accessibility

Designs should consider:

- Color contrast
- Readable typography
- Keyboard navigation where applicable
- Alternative text when appropriate
- Responsive layouts
- Inclusive imagery

Accessibility improves usability for everyone.

---

# Brand Consistency

Every design should respect:

- Official logos
- Approved colors
- Typography standards
- Brand voice
- Existing design systems

Do not invent branding unless requested.

---

# Production Quality

Deliverables should be ready for real-world use.

Avoid:

- Placeholder graphics
- Misaligned elements
- Inconsistent spacing
- Pixelated assets
- Low-resolution exports
- Unbalanced layouts

---

# Design Review Checklist

Before completion, verify:

- Clear hierarchy
- Consistent spacing
- Brand alignment
- Readability
- Accessibility
- Production quality
- Appropriate imagery
- Obvious call to action (when applicable)

---

# Final Design Test

Ask:

> Does this make the user's objective easier to understand and accomplish?

If the answer is no, revise the design.
