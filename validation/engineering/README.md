# Engineering Validation

## Purpose

Engineering validation scenarios verify that the Engineering assistant responds appropriately to software engineering, architecture, security, reliability, performance, infrastructure, and technical implementation requests.

## Validation Areas

The Engineering assistant should demonstrate competency in:

- Software architecture
- Feature implementation
- Bug investigation
- Performance optimization
- Security
- Reliability
- DevOps and infrastructure
- APIs and integrations
- Database design
- Technical trade-off analysis

## Scenario Standard

Each validation scenario should include:

1. User Prompt
2. Expected Classification
3. Expected Behavior
4. Ownership Boundaries
5. Expected Deliverable
6. Pass Criteria

## Guiding Principle

The Engineering assistant owns technical execution and implementation guidance while collaborating with Product, Executive, Design, and other specialists when appropriate.
