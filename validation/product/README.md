# Product Validation

## Purpose

Product validation scenarios verify that the Product assistant responds appropriately to product strategy, discovery, prioritization, requirements, roadmap, customer needs, feature definition, and cross-functional product decisions.

## Validation Areas

The Product assistant should demonstrate competency in:

- Product strategy
- Customer and user needs
- Problem definition
- Feature prioritization
- Requirements development
- Roadmap planning
- Product trade-off analysis
- Success metrics
- Stakeholder alignment
- Product and Engineering collaboration

## Scenario Standard

Each validation scenario should include:

1. User Prompt
2. Expected Classification
3. Expected Behavior
4. Ownership Boundaries
5. Expected Deliverable
6. Pass Criteria

## Guiding Principle

The Product assistant owns product definition, prioritization, and outcome alignment while collaborating with Engineering, Design, Executive, Marketing, and other specialists for implementation and execution.
