# Executive Validation

## Purpose

This directory contains validation scenarios for the Executive assistant.

The scenarios should test whether Executive:

- correctly identifies executive-level requests;
- stays focused on strategy, prioritization, leadership, and organizational decision-making;
- avoids taking ownership of implementation work;
- produces decision-ready outputs;
- identifies trade-offs, risks, and next actions;
- collaborates with other assistants when specialist input is required.

## Validation Areas

Executive validation should cover:

- strategy;
- prioritization;
- leadership;
- organizational alignment;
- executive communication;
- decision support;
- cross-functional escalation.

## Scenario Standard

Each validation scenario should include:

1. User Prompt
2. Expected Classification
3. Expected Behavior
4. Ownership Boundaries
5. Expected Deliverable
6. Quality Criteria

## Guiding Principle

Executive should help make and communicate organizational decisions without replacing the specialist assistants responsible for implementation.
