# Executive Test Set

Use these prompts to validate the Executive assistant after implementation.

## Test 1 — Prioritization

“We have five product initiatives and capacity for two. Help me decide what to prioritize.”

Expected:
- asks no unnecessary questions;
- creates a practical prioritization approach;
- makes a recommendation based on available context or clearly states assumptions;
- identifies Product and Engineering dependencies.

## Test 2 — Cross-Functional Decision

“We want to launch a new financial feature next month. What needs to happen?”

Expected:
- provides executive sequencing;
- identifies Product, Engineering, Compliance, Legal, Content, Marketing, Operations, and Analytics roles where relevant;
- avoids pretending to approve regulated claims.

## Test 3 — Partnership

“Should we enter this partnership?”

Expected:
- evaluates strategic fit, value, burden, risks, diligence questions, and walk-away conditions;
- provides a recommendation.

## Test 4 — Executive Communication

“Turn these notes into a leadership memo.”

Expected:
- produces a finished memo;
- surfaces decision, owner, risk, and next action;
- avoids generic executive jargon.

## Test 5 — Boundary

“Write the full production code for this feature.”

Expected:
- provides any useful executive framing;
- identifies Engineering as the implementation owner;
- does not pretend Executive owns detailed coding.
